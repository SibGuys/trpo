 Тип оценки
Проект разработки, т.к. оценивается количество функциональности поставляемой пользователям в первом релизе  проекта.
 
 Область оценки
Все разрабатываемые функции ( проект разработки)

 Подсчёт функциональных точек, связанных с данными  
Объект “Работник” -  данные работника, которые ему необходимо внести работнику (ILF):

Группы данных: 
ФИО, дата выдачи карточки, страна работы. (3 RET)

Неповторяемые уникальные поля данных:
Фамилия, Имя, Отчество,  страна работы, дата (5 DET).

Объект “Отчёт” - отчёт, который составляет администратор (ILF):

Группа данных:
Тип отчёта(1 RET)

Неповторяемые уникальные поля данных:
Отчёт о времени работы, отчёт об отпускных днях, очтёт о заработной плате, отчёт о переработках. (4 DET)

Объект “Учёт рабочего времени” - карточка учёта рабочего времени для работников по часовой ставке (ILF):

Группа данных:
Дата, количество часов, тариф (3 RET).

Неповторяемые уникальные поля данных:
Число месяца, месяц, год, часы, коэффициент тарифа, количество переработанных часов (6 DET).

Объект “Заказы на поставку” - информация, которую предоставляют  работники, получающие фиксированный оклад (ILF) 

Группа данных:
Дата, количество проданного товара, тип товара(3 RET).

Неповторяемые уникальные поля данных:
Число месяца, месяц, год, количество, тип (5 DET).

Объект “Способ оплаты” - тип оплаты, выбранный работником (ILF)

Группа данных:
Тип оплаты (1 RET)

Неповторяемые уникальные поля данных:
Чек по почте, наличные, на счёт в банк (3 DET).

Объект “Администратор” - работник, который пишет отчёты, ведёт информацию о служащих (ILF)

Группа данных:
работник, тип оплаты (2 RET)

Неповторяемые уникальные поля данных:
Фамилия, Имя, Отчество, страна, чек, перевод на счёт в банк, наличные.(7 DET)


Объект “Существующая БД” - прошлая БД, которую компания сохранила из соображений экономии (EIF): 

Группа данных:
информация о прошлых проектах, тарифы (2 RET)

Неповторяемые уникальные поля данных:
Название проекта, дата начала, дата завершения, бюджет проекта, задействованные работники, тариф региона (6 DET) 

Объект “Отпуска работников” - даты отпусков работников, которые может отредактировать администратор (ILF)

Группа данных:
ФИО работника (ID), дата отпуска (2 RET)

Неповторяемы уникальные поля данных:
Фамилия, имя, отчество, дата начала отпуска, дата окончания отпуска (5 DET)

Объект “Заработная плата” - информация о сумме выплаты, комиссионные (ILF): 

Группа данных:
ФИО работника (ID), сумма выплаты, процент комиссионных, товар (3 RET)

Неповторяемые поля данных:
Фамилия, имя, отчество, сумма выплат за смену, сумма выплат за месяц, процент комиссионных, количество проданного товара (6 DET)

4.Подсчёт функциональных точек, связанных с транзакциями:

Транзакция 1 “Авторизация работника”
Три поля ввода для ФИО. Кнопка выбора страны из списка. При нажатии на кнопку “Рабочее время” открывается диалоговое окно, в котором пользователь вводит дату выдачи карточки. Тариф заполнен автоматически(зависит от выбора страны). Транзакция типа EI, т.к. происходит внешний ввод данных.

Количество DET:
Поля: Фамилия, Имя, Отчество, страна, дата (всего 5 уникальных данных).
DET: 7

Количество FTR:
ILF: Работник.
FTR: 1
Тип транзакции:
Ввод данных и обработка через диалоговые окна, то есть это EI (External Input).
Сложность транзакции:
Low
UFP: 3
Транзакция 2: "Создание отчёта"
Поля для выбора типа отчёта: "Отчёт о времени работы", "Отпускные дни", "Заработная плата", "Переработки". Администратор выбирает тип отчёта из 4 вариантов (1 поле ввода).
Генерация шаблона отчёта (1 выходное поле).
Количество DET:
Поля: Поле выбора типа отчёта, 4 вида отчётов.
DET: 5 
Количество FTR:
ILF: Отчёт
FTR: 1
Тип транзакции:
Транзакция является EQ, т.к. при выборе типа отчёта, генерируется шаблон. Происходит одновременный ввод и вывод.
Сложность транзакции: 
Low.
UFP: 4
Транзакция 3: "Запись учёта рабочего времени"
Описание:
Пользователь вводит количество отработанных часов. В диалоговом окне выбирает дату.
DET:
Дата (число, месяц, год), количество часов.
DET: 4
FTR:
ILF: Учёт рабочего времени.
FTR: 1
Тип транзакции:
Это EI.
Сложность транзакции: Low.
UFP = 3
Транзакция 4: "Запись о заказах на поставку"
Пользователь вводит информацию о заказах на поставку. Указывает дату в соответствующем окне. Выбирает тип товара из списка, количество проданного товара по типу. После заполнения всех полей генерируется процент комиссионных, который высчитывается исходя из типа проданного товара и его количества.
DET:
Дата(число, месяц, год), количество, тип товара, процент комиссии.
DET: 6
FTR:
ILF: Заказы на поставку.
FTR: 1
Тип транзакции:
Это EQ.
Сложность транзакции: Low.
UFP = 3
Транзакция 5: "Выбор способа оплаты"

DET:
Чек, перевод на счёт в банк, наличные.
DET: 4
FTR:
ILF: Способ оплаты.
FTR: 1
Тип транзакции:
Это EI.
Сложность транзакции: Low.
UFP = 3.
Транзакция 6: "Авторизация администратора”
Администратор обладает полными правами на редактирование. При начале работы он вводит данные как в транзакции “авторизация работника”. После этого администратор может перейти к списку работников и изменить их данные, указанные при авторизации.
DET:
Фамилия, Имя, Отчество, страна, дата, ФИО работника, страна работника, дата указанная работником.
DET: 10
FTR:
ILF: Авторизация работника, авторизация администратора.
Итого FTR: 2.
Тип транзакции:
Это EI.
Сложность транзакции: Average.
Для Average сложности EI транзакции UFP = 4.
